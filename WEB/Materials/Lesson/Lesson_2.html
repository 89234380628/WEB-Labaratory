Урок 2. Знакомство со структурой HTML

Перед тем, как начать изучать сами HTML теги, а в последующем CSS свойства, необходимо разобраться в самой структуре HTML страницы.

Структура HTML страницы

<!DOCTYPE html>
                        <html lang="ru">
                        <head>
                            <meta charset="UTF-8">
                            <meta http-equiv="X-UA-Compatible" content="IE=edge">
                            <meta name="viewport" content="width=device-width, initial-scale=1.0">
                            <title>Title for page</title>
                        </head>
                        <body>
                            <!--Комментарий-->
                            <h1>Заголовок 1 уровня</h1>
                            <p>Первый абзац</p>
                            <p>Второй абзац</p>
                        </body>
                        </html>
<!DOCTYPE html>

Элемент <!DOCTYPE> предназначен для указания типа текущего документа.

Это необходимо, чтобы браузер понимал, как следует интерпретировать текущую веб-страницу, поскольку >HTML существует в нескольких версиях, кроме того, имеется XHTML, похожий на HTML, но различающийся с ним по синтаксису. Чтобы браузер «не путался» и понимал, согласно какому стандарту отображать веб-страницу и необходимо в первой строке кода задавать <!DOCTYPE>.
Основные типы документов DOCTYPE
DOCTYPE Описание
HTML 5
<!DOCTYPE html> Подходит для всех версий HTML документов
HTML 4.01
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd"> Строгий синтаксис HTML
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN" "http://www.w3.org/TR/html4/loose.dtd"> Переходный синтаксис HTML
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Frameset//EN" "http://www.w3.org/TR/html4/frameset.dtd"> В HTML-документе применяются фреймы
XHTML 1.1
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd"> Разработчики XHTML 1.1 предполагают, что он постепенно вытеснит HTML. Никакого деления на виды это определение не имеет, синтаксис один и подчиняется четким правилам
XHTML 1.0
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd"> Строгий синтаксис XHTML
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd"> Переходный синтаксис XHTML
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd"> Документ написан на XHTML и содержит фреймы
Разница между строгим и переходным описанием документа состоит в различном подходе к написанию кода документа. Строгий HTML требует жесткого соблюдения спецификации HTML и не прощает ошибок. Переходный HTML более «спокойно» относится к некоторым огрехам кода, поэтому этот тип в определенных случаях использовать предпочтительнее.

Например, в строгом HTML и XHTML непременно требуется наличие тега <title>, а в переходном HTML его можно опустить и не указывать. При этом помним, что браузер в любом случае покажет документ, независимо от того, соответствует он синтаксису или нет. Подобная проверка осуществляется при помощи валидатора и предназначена в первую очередь для разработчиков, чтобы отслеживать ошибки в документе.

Важно:

Часто можно встретить код HTML вообще без использования <!DOCTYPE>, веб-страница в подобном случае все равно будет показана.
Тем не менее, может получиться, что один и тот же документ отображается в браузере по-разному при использовании <!DOCTYPE> и без него. Кроме того, браузеры могут по-своему показывать такие документы, в итоге страница «рассыплется», т.е. будет отображаться совсем не так, как это требуется разработчику.
Чтобы не произошло подобных ситуаций, всегда добавляйте <!DOCTYPE> в начало документа.

<html lang="...">...</html>

Первый тег <html> определяет начало HTML - файла.

Второй (закрывающий) тег показывает конец HTML - файла.

<head>...</head>

Первый тег <head> определяет начало блока заголовка HTML - файла.

Второй (закрывающий) тег показывает конец блока заголовка HTML - файла.

Важно:

Данный блок заголовка содержит только информацию об HTML - файле и его содержимое не будет показано на самой странице.

<meta charset="UTF-8">

Содержит информацию о кодировке которая используется на данной WEB - странице

Важно:

Зачастую все современные версии браузеров без труда могут определить кодировку на WEB - странице и правильно отобразят кирилицу алфавита.


Но менее новые версии, такой функцией могут не обладать или бывают случае, что браузер на распознал кодировку и в таких случаях, если не была указана информация об используемой кодировки на странице, то вместо ожидаемой кирилице, страница может вывести непонятную "аброкадабру".

Основные кодировки которые используют на WEB - страницах

Кодировка Описание
ASCII Однобайтовая кодировка, в которую изначально заложено всего 128 символов: буквы латинского алфавита, арабские цифры и т.д.

ISO 8859-1 Расширенная латиница, включающая символы большинства западноевропейских языков (английский, датский, ирландский, исландский, испанский, итальянский, немецкий, норвежский, португальский, ретороманский, фарерский, шведский, шотландский (гэльский) и частично голландский, финский, французский), а также некоторых восточноевропейских (албанский) и африканских языков (африкаанс, суахили).
ISO 8859-2 Расширенная латиница, включающая символы центральноевропейских и восточноевропейских языков (боснийский, венгерский, польский, словацкий, словенский, хорватский, чешский).
ISO 8859-3 Расширенная латиница, включающая символы южноевропейских языков (мальтийский, турецкий и эсперанто).
ISO 8859-4 Расширенная латиница, включающая символы североевропейских языков (гренландский, эстонский, латышский, литовский и саамские языки).
ISO 8859-5 Кириллица, включающая символы славянских языков (белорусский, болгарский, македонский, русский, сербский и частично украинский).

ISO 8859-13 ИСпользуется для балтийских языков

ISO 8859-15 Редко используемые символы заменены на необходимые для полной поддержки финского, французского и эстонского языков. Так же был добавлен знак евро.

<meta http-equiv="X-UA-Compatible" content="IE=edge">

Содержит информацию о том, как необходимо обрабатывать вашу WEB - страницу старым версиям браузеров

<title>Title for page</title>

Важнейший HTML - компонент сайта, который влияет на ранжирование и CTR (кликабельность).

Важно:

Тег <title> является обязательным и должен непременно присутствовать в коде документа

<body>...</body>

Тело документа <body> предназначено для размещения тегов и содержательной части веб-страницы
